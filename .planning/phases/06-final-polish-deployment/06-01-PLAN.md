# Plan 06-01: Write Firestore/Storage security rules

Configure strict security rules for database and storage access.

## Proposed Changes
- Create `firestore.rules` containing security guidelines for the database.
- Limit users so they can only read and write documents belonging to their own `userId` paths:
  ```
  rules_version = '2';
  service cloud.firestore {
    match /databases/{database}/documents {
      match /users/{userId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
        
        match /meals/{mealId} {
          allow read, write: if request.auth != null && request.auth.uid == userId;
        }
      }
    }
  }
  ```
- Create `storage.rules` setting up similar constraints for meal images stored in Cloudinary (or Firebase Storage if used, though currently images upload to Cloudinary).
